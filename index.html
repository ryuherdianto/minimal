Migrasi Server Zimbra dan Transfer Account Zimbra – The Best Way

1. Pendahuluan

Project Zimbra tidak memiliki dokumentasi sebuah cross migration atau transfer account yang baik. Yang mereka sarankan adalah dengan cara meng-copy folder /opt/zimbra ke server baru anda. Tetapi bagaimana jika file-file tersebut sudah terinfeksi dengan rootkit atau malicious script lainnya?, maka server baru kita akan ter-compromised. Jadi jangan pernah sync atau copy keseluruhan direktori dari folder instalasi zimbra kita. Zimbra juga menganjurkan kita untuk meng-upgrade production server kita ke versi terakhir sebelum melakukan migrasi. Tetapi dengan cara yang tidak tepat kita dapat teresiko kehilangan data. Dengan cara berikut anda dapat: Me-migrasi kan zimbra dari satu Operating System ke yang lainnya. Me-migrasi kan account zimbra antar hardware berbeda dan konfigurasi Operating systems yang berbeda. Tidak akan ada interupsi di sisi production server seperti software upgrade atau service disable. Migrasi zimbra dari versi lama ke versi yang lebih baru Cross migration Zimbra tanpa harus meng-copy keseluruhan direktori.

2. Requirement

Anda memerlukan server lama anda dengan akun zimbra dan sebuah server baru dengan OS yang anda inginkan. Jangan membuat menciptakan konfigurasi atau setting apapun pada server baru anda. Pastikan anda mengkonfigurasikan hostname server baru anda sama dengan server yang lama. Server lama Membutuhkan login ssh root Membutuhkan login admin zimbra Kapasitas HDD yang cukup untuk menyimpan backup Server baru Harus di install dengan stable zimbra terbaru Membutuhkan login ssh root Membutuhkan login zimbra admin Kapasitas HDD yang cukup untuk menyimpan backup

3. Presetup

You need to setup an ssh key from the new server’s root account to the old server’s root account. Reduce the TTL of MX records of your domain to 500 seconds . So that you can easily switch the domain’s IP after migration. Please remember to schedule the migration task on non peek hours. Create a directory in both new and old server into which we store all required files and data for doing the migration
